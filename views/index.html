Что нужно сделать?

Необходимо создать мини-веб-приложение, которое бы позволяло выбрать населённый пункт и посмотреть его расположение на карте

Звучит немного расплывчато, не находите?

Окей, давайте подробнее

Логика работы

Необходимо сделать три input'а, на каждый навесить autocomplete (из jquery ui), который будет брать данные по ajax.
Первый input будет отвечать за субъекты РФ, второй — за районы, третий – за населённые пункты. Изначально второй и третий input'ы неактивны (или скрыты).
Пользователь набирает название субъекта в первый input, ему предлагаются различные варианты для выбора.
Он выбирает какой-либо субъект, второй input становится активным (появляется).
Затем пользователь набирает название района во второй input, ему также показываются различные варианты (районы, которые относятся к этому субъекту).
Аналогично происходит с третьим input'ом и населёнными пунктами.
При выборе населённого пункта происходит запрос на сервер, который возвращает объект с геоданными этого населённого пункта (координаты для карты), и отображается карта (с помощью yandex maps api).
Центр карты должен совпадать с координатами города.

REST API

Сервер — https://cdu-task.herokuapp.com, все запросы — GET
  • /getRegionsList
    Параметры: name — string
    Возвращается массив объектов (субъектов РФ) вида 
    { "id": number , "name": string }
  • /getDistrictsByRegionID
    Параметры: id — number, name — string
    Возвращается массив объектов (района субъекта РФ с заданным "id") вида 
    { "id": number , "name": string }
  • /getCitiesByDistrictID
    Параметры: id — number, name — string
    Возвращается массив объектов (населённые пункты района с заданным "id") вида 
    { "id": number , "name": string }
  • /getCityByID
    Параметры: id — number
    Возвращается объект (населённый пункт) вида 
    { "id": number , "name": string , "geoData": [number, number] }
